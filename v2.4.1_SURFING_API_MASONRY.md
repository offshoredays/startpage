## 📚 공공데이터포털 API 키 발급 가이드

### 🎯 **단계별 완벽 가이드**

#### 1단계: 회원가입 & 로그인
```
https://www.data.go.kr
→ 우측 상단 "로그인" 클릭
→ 회원가입 (카카오/네이버 간편가입 가능)
```

#### 2단계: 서핑지수 API 찾기
```
방법 A: 검색창에 "서핑지수" 입력
방법 B: 직접 접속
        https://www.data.go.kr/data/15097079/openapi.do
```

#### 3단계: 활용신청
```
"활용신청" 버튼 클릭
→ 신청 목적: "개인 스타트페이지 서핑 정보 표시"
→ 상세기능: "실시간 파도/서핑지수 확인"
→ 신청하기 클릭
```

#### 4단계: 승인 확인 (보통 즉시!)
```
마이페이지 → 오픈API → 개발계정
→ 서핑지수 API 찾기
→ 상태: "승인" 확인
```

#### 5단계: 인증키 복사
```
서핑지수 API 클릭
→ "일반 인증키 (Encoding)" 복사
   (예: f8906336e1234567abcd...)
```

#### 6단계: 스타트페이지 설정
```
설정 (⚙️) → 서핑지수 API 설정
→ API 인증키: 붙여넣기
→ 관측소 코드: TW_0062 (울산 기본값)
→ 저장
```

```javascript

  "waveDirection": "NE",  // 파도 방향

## 🧪 테스트 방법

### 1. API 키 없을 때
```
1. Ctrl+Shift+R (강제 새로고침)
2. 날씨 위젯 확인
3. "🌊 API 키 필요" 표시됨 ✅
```

### 2. API 키 발급 후
```
1. 공공데이터포털에서 키 발급
2. 설정에 키 입력 → 저장
3. 날씨 위젯 확인
4. "🌊 1.2m NE 🏄 좋음" 표시 ✅
```

### 3. Masonry 레이아웃 확인

1. 페이지 새로고침

```

## 🐛 트러블슈팅

### Q: API 키를 입력했는데도 "API 키 필요"가 나와요
**A: 콘솔 (F12) 확인**
```javascript
console.log('API Key:', localStorage.getItem('settings'));
// surfingApiKey가 저장되었는지 확인

// 강제 재로드
await loadWeatherData(app);
```

### Q: "INVALID_REQUEST_PARAMETER_ERROR" 오류
**A: 관측소 코드 확인**
```
설정 → 관측소 코드 → TW_0062 (울산)
또는 API 문서에서 정확한 코드 확인
```

### Q: "SERVICE_KEY_IS_NOT_REGISTERED" 오류
**A: API 키 재확인**
```
1. 공공데이터포털 → 마이페이지 → 오픈API
2. "일반 인증키 (Encoding)" 다시 복사
3. 설정에 다시 붙여넣기
```

### Q: 카테고리가 이상하게 배치돼요
**A: 브라우저 캐시 삭제**
```
Ctrl+Shift+Delete → 캐시 삭제
또는 Ctrl+Shift+R (강제 새로고침)
```
