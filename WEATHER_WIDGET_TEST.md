## ✅ 테스트 시나리오 (5분)

### 테스트 1: 기본 설정 ⚙️
**목표**: API 키 입력 후 데이터 표시 확인

1. **날씨 위젯 설정 열기**
   - 날씨 위젯의 ⚙️ 아이콘 클릭
   
2. **API 키 입력**
   - "공공데이터 포털 API 키" 입력
   - 해수욕장: "울산 일산해수욕장" 선택
   
3. **표시 정보 선택**
   - ✅ 기온 (체크됨)
   - ✅ 수온 (체크됨)
   - ✅ 파도 높이 (체크됨)
   - ❌ 파도 방향 (체크 해제)
   - ❌ 파도 주기 (체크 해제)
   - ✅ 풍속 (체크됨)
   - ❌ 바람 방향 (체크 해제)
   - ✅ 파도 상태 (체크됨)
   
4. **저장 클릭**

**예상 결과**:
- 위젯에 `15°` 같은 기온 표시
- 위치: `울산`
- 바람: `🌬️ 3.2m/s`
- 파도: `🌊 0.5m (낮음)`

5. **콘솔 확인 (F12)**
   ```javascript
   🏄 서핑지수 API 요청: https://www.khoa.go.kr/...
   🏄 응답 (raw): <response>...
   🏄 파싱된 데이터: { temp: "15.5", wavHgt: "0.5", ... }
   💾 날씨 설정 저장: { apiKey: "✅ 설정됨", beach: "102", ... }
   ```

### 테스트 2: 표시 정보 커스터마이징 🎨
**목표**: 원하는 정보만 선택해서 표시

1. **설정 다시 열기** (⚙️ 클릭)

2. **모든 옵션 활성화**
   - ✅ 기온
   - ✅ 수온
   - ✅ 파도 높이
   - ✅ 파도 방향
   - ✅ 파도 주기
   - ✅ 풍속
   - ✅ 바람 방향
   - ✅ 파도 상태

3. **저장 후 확인**
   - 바람: `🌬️ 3.2m/s 북동`
   - 파도: `🌊 0.5m 동 (낮음)`

4. **최소 정보만 표시**
   - ✅ 기온
   - ❌ 수온
   - ✅ 파도 높이
   - ❌ 나머지 모두 체크 해제

5. **저장 후 확인**
   - 바람: `🌬️ --`
   - 파도: `🌊 0.5m`

**예상 결과**: 체크한 정보만 표시, 체크 해제한 정보는 `--` 또는 표시되지 않음

### 테스트 5: 동기화 확인 🔄
**목표**: GitHub Gist에 설정이 저장되는지 확인

1. **콘솔 실행** (F12)
   ```javascript
   // 현재 설정 확인
   console.log('🏄 날씨 설정:', {
       apiKey: app.settings.surfingApiKey ? '✅' : '❌',
       beach: app.settings.surfingBeachNum,
       showTemp: app.settings.showTemp,
       showWave: app.settings.showWave
   });
   ```

2. **수동 푸시 (선택사항)**
   ```javascript
   await app.githubSync.pushData();
   ```

3. **다른 기기에서 풀**
   ```javascript
   await app.githubSync.pullData(true);
   ```

4. **설정 확인**
   - 날씨 위젯에 같은 데이터 표시

## 🐛 문제 해결

### API 키가 작동하지 않음
- **증상**: `API 키 필요` 표시
- **해결**:
  1. 공공데이터 포털에서 승인 확인
  2. "일반 인증키 (Encoding)" 복사 (Decoding 아님!)
  3. 설정 다시 입력

### 데이터가 표시되지 않음
- **증상**: `--°`, `--` 표시
- **해결**:
  1. F12 콘솔 확인
  2. 🏄 API 요청 URL 확인
  3. 응답 데이터 확인 (XML 또는 JSON)

### 설정이 저장되지 않음
- **증상**: 새로고침 후 설정 초기화
- **해결**:
  1. 콘솔에서 `💾 날씨 설정 저장` 로그 확인
  2. `localStorage.getItem('settings')` 확인
  3. GitHub Sync 설정 확인
