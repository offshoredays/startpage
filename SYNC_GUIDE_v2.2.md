# 🎯 GitHub 동기화 완벽 가이드 (v2.2)

## ✨ 새로운 기능: 자동 Gist 검색!

이제 두 번째 기기에서 **같은 Token만 입력하면 자동으로 기존 Gist를 찾아줍니다!**

---

## 📖 사용 방법

### 1️⃣ 첫 번째 기기 (집)

#### Step 1: Token 발급
https://github.com/settings/tokens/new?description=Startpage%20Sync&scopes=gist

#### Step 2: 연결
```
1. 스타트 페이지 → 설정(⚙️)
2. GitHub Token 입력
3. "GitHub 연결하기" 클릭
4. "새 Gist를 생성했습니다" 확인
```

#### Step 3: 북마크 추가
```
북마크를 추가하면 자동으로 5분마다 동기화됩니다.
```

---

### 2️⃣ 두 번째 기기 (회사) - 자동 검색! ✨

#### Step 1: 같은 Token 입력
```
1. 스타트 페이지 → 설정(⚙️)
2. 첫 번째 기기와 **정확히 같은 Token** 입력
3. "GitHub 연결하기" 클릭
```

#### Step 2: 기존 Gist 자동 발견!
```
팝업 메시지:
"기존 Startpage Gist를 발견했습니다!
✅ 확인: 기존 데이터를 불러옵니다 (권장)
❌ 취소: 새로운 Gist를 생성합니다"

→ "확인" 클릭!
```

#### Step 3: 완료!
```
"✅ 기존 데이터를 성공적으로 불러왔습니다!"

이제 모든 북마크가 동기화되었습니다! 🎉
```

## ❓ FAQ

### Q1: 두 번째 기기에서 "기존 Gist를 발견했습니다" 메시지가 안 뜨면?

**A**: Token이 다르거나 첫 번째 기기에서 아직 Gist가 생성되지 않았을 수 있어요.

**해결**:
1. 첫 번째 기기에서 북마크를 추가해보세요 (Gist 생성 트리거)
2. F12 콘솔에서 확인:
   ```javascript
   console.log('Gist ID:', localStorage.getItem('github_gist_id'));
   ```
3. Gist ID가 있으면 성공!

### Q2: "새 Gist를 생성했습니다"가 두 번 뜨면?

**A**: 실수로 두 개의 Gist를 만든 거예요.

**해결**:
1. https://gist.github.com/ 접속
2. "Secret" 탭에서 중복된 Gist 삭제
3. 남은 Gist의 ID 확인
4. 양쪽 기기에서:
   ```javascript
   localStorage.setItem('github_gist_id', '여기에_Gist_ID');
   location.reload();
   ```

### Q3: 자동 동기화가 너무 느린데?

**A**: 기본 5분 간격이에요. 더 빠르게 하려면:

```javascript
// 콘솔에서 실행 (1분마다)
app.githubSync.startAutoSync(1);
```

또는 "지금 동기화" 버튼 사용!

### Q4: 동기화가 충돌하면?

**A**: 최신 타임스탬프를 우선합니다.

**권장**:
- 한 번에 한 기기에서만 작업
- 기기 전환 전 "지금 동기화" 클릭
- 새 기기에서 "클라우드에서 가져오기" 클릭

## 🔒 보안 팁

1. **Token 관리**
   - 비밀번호 관리자에 저장 (1Password, Bitwarden 등)
   - 공용 PC에서는 사용 후 "연결 해제"

2. **Gist는 Private**
   - 자동으로 비공개 저장
   - https://gist.github.com/에서 확인 가능

3. **Token 권한**
   - `gist` 권한만 필요 (다른 권한 불필요)
